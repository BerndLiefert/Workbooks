Vergleiche

die Vergleiche auf Gleichheit führen bei primitiven Datentypen immer zu den "richtigen" Ergebnissen

int x = 100;
int y = 100;
if(x == y)
//gleich

die Vergleiche auf Gleichheit bei Objekten müssen differenziert betrachtet werden

String s1 = new String("Hallo Welt")
String s2 = new String("Hallo Welt")
if(s1 == s2)
//ungleich, == vergleicht die Objektreferenzen
if(s1.equals(s2))
//gleich, equals vergleicht je nach Implementierung

Lösung:

Wie muss equals() implementiert werden? "equals contract"
  1. Reflexivität: jedes Objekt liefert im Vergleich mit sich selbst TRUE
  2. Symetrie: wenn x "==" y TRUE ist, dann muss auch y "==" x TRUE sein
  3. Transitivität: wenn x "==" y TRUE ist und y "==" z TRUE ist, dann muss auch x "==" z TRUE sein
  4. Konsistenz: x "==" y muss auch bei wiederholter Ausführung TRUE/FALSE sein
  5. für alle x != 0 -> x "==" null muss FALSE sein

Fehlerbehandlung mit Exceptions

die Exceptions treten zur Laufzeit auf

Exceptions sind Objekte der Klasse Exception bzw. von deren Subklassen

Exceptions sind throwable

Arten von Exceptions

  1. checked exception
    • Ausnahmen, die vorhersehbar sind
    • Ausnahmen, die durch den Programmierer selbst behandelt werden können (catch)
  2. error
    • Ausnahme, die nicht durch den Programmierer behandelt werden können
  3. runtime exception
    • können nicht gefangen werden
    • "programminterne" Ausnahmen
    • Bsp.:
      • a) Division druch 0
      • b) NullPointerException
