Vergleiche equals()

die Vergleiche auf Gleichheit führen bei primitiven Datentypen immer zu den "richtigen" Ergebnissen

int x = 100;
int y = 100;
if(x == y)
//gleich

die Vergleiche auf Gleichheit bei Objekten müssen differenziert betrachtet werden

String s1 = new String("Hallo Welt")
String s2 = new String("Hallo Welt")
if(s1 == s2)
//ungleich, == vergleicht die Objektreferenzen
if(s1.equals(s2))
//gleich, equals vergleicht je nach Implementierung

Lösung:

Wie muss equals() implementiert werden? "equals contract"
  1. Reflexivität: jedes Objekt liefert im Vergleich mit sich selbst TRUE
  2. Symmetrie: wenn x "==" y TRUE ist, dann muss auch y "==" x TRUE sein
  3. Transitivität: wenn x "==" y TRUE ist und y "==" z TRUE ist, dann muss auch x "==" z TRUE sein
  4. Konsistenz: x "==" y muss auch bei wiederholter Ausführung TRUE/FALSE sein
  5. für alle x != 0 -> x "==" null muss FALSE sein

Das Collection-Framework

"Rahmenwerk" für Datenstrukturen (dynamisch)

im Collection-Framework finden wir

  1. Interfaces (Verhalten von Datenstrukturen)
  2. Implementierungen (Klassen, die die Interfaces implementieren)
  3. Algorithmen, die nützliche Funktionen für Datensammlungen bereitstellen

Vorteile:

  • reduziert den Programaufwand
  • verbessert die Qualität der Software
  • verbessert die Zusammenarbeit mit anderen durch
    • wohldefinierte Schnittstellen

Collection Interface: die Wurzel der meisten Implementierungen

Sammlung von Objekten, die als Elemente bezeichnet werden

in der Doku , e -> .add()

Set: Menge von Objekten ohne Duplikate

List: Liste von Objekten, die Duplikate enthalten darf und sortiert ist

Queue: Warteschlange, FIFO "wer zuerst kommt, mahlt zuerst"

Deque: eine double ended queue, Warteschlange, FIFO (First In – First Out), LIFO (Last In - First Out)

Map: Schlüsselwert -> Wert
key -> value
